Walking Through a Simple Spring MVC CRUD Application

This tutorial will walk you through a basic CRUD application using Spring MVC with Force.com as the underlying database. The application lets you browse and query a few related entities and create and update those entities.

The tutorial is based on the VMforce Music Library sample app and instead of pasting long stretches of code into this tutorial we will just reference the code on github.

Starting Point

The Musiclib application is based on the Force.com Spring MVC archetype, which again is based on the basic Spring MVC 3.0.5 project template. When you build your own application, as you did in the quick start guide, you can start from this same archetype and avoid having to manually set up Spring and JPA configuration files. The archetype generates the following:

  • Java package directories for model, controller and service classes
  • Spring application context and JPA persistence.xml files
  • Basic JSPs with a simple stylesheet and template setup
  • A single integration test
  • And of course, the pom.xml file that’s used to build the project and manage dependencies
